1. Авторизация пользователя
Пользователь вводит логин и пароль в клиентском приложении (JavaFX).
Клиент отправляет запрос авторизации на сервер (Spring REST API).
REST API обращается к Spring Security для проверки данных.
После успешной авторизации пользователю возвращается JWT-токен, используемый в дальнейших запросах.

2. Создание и анализ экономической модели
Пользователь создаёт новую модель, указывая её название, параметры и загружая документацию.
Клиент отправляет запрос на сервер с данными новой модели.
Сервер сохраняет модель в базу данных.
Сервер отправляет документацию в AI-сервис (OpenAI API) для анализа и извлечения фактических данных.
AI возвращает фактические данные, сервер сохраняет их в базе данных.
Пользователь получает модель и результаты AI-анализа и может сравнить модель с реальными показателями.

3. Запуск вычислений с использованием кеширования
Пользователь инициирует расчёты, выбирая модель и задавая параметры.
Клиент отправляет запрос на сервер.
Сервер проверяет кеш (Spring Cache):
Если результат есть в кеше, он сразу возвращается пользователю.
Если результата нет, сервер извлекает данные модели из базы данных, проводит вычисления и сохраняет результат в кеш.
